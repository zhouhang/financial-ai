# 启动三个服务 - 聊天对话记录

> 本文档记录关于 Financial AI 项目启动三个核心服务的完整信息，便于后续对话继承和快速查阅。

---

## 三个服务概览

| 服务 | 端口 | 地址 | 说明 |
|------|------|------|------|
| **finance-mcp** | 3335 | http://localhost:3335 | MCP 工具服务（数据整理、对账） |
| **data-agent** | 8100 | http://localhost:8100 | API 服务（对话、文件上传等） |
| **finance-web** | 5173 | http://localhost:5173 | 前端用户界面 |

---

## 一键启动（推荐）

```bash
cd /Users/kevin/workspace/financial-ai
./START_ALL_SERVICES.sh
```

脚本会自动：
1. 停止旧服务（3335、8100、5173 端口）
2. 启动 finance-mcp
3. 启动 data-agent
4. 启动 finance-web
5. 验证服务状态
6. 输出访问地址和日志路径

---

## 手动启动（三个终端）

**终端 1 - finance-mcp:**
```bash
cd /Users/kevin/workspace/financial-ai
source .venv/bin/activate
cd finance-mcp
python unified_mcp_server.py
```

**终端 2 - data-agent:**
```bash
cd /Users/kevin/workspace/financial-ai
source .venv/bin/activate
cd finance-agents/data-agent
python -m app.server
```

**终端 3 - finance-web:**
```bash
cd /Users/kevin/workspace/financial-ai/finance-web
npm run dev
```

---

## 停止服务

```bash
lsof -ti:3335,8100,5173 | xargs kill -9
```

---

## 日志位置

使用 `START_ALL_SERVICES.sh` 启动时，日志在 `logs/` 目录：

```bash
tail -f logs/finance-mcp.log    # finance-mcp 日志
tail -f logs/data-agent.log     # data-agent 日志
tail -f logs/finance-web.log    # finance-web 日志
```

---

## 前置条件

- **Python 虚拟环境**: 项目根目录 `.venv`，需先 `source .venv/bin/activate`
- **Node.js**: finance-web 需要 `npm run dev`
- **端口可用**: 3335、8100、5173 未被占用

---

## 常见问题

1. **端口被占用**: `lsof -i:3335` 查看占用，`kill -9 <PID>` 结束进程
2. **依赖缺失**: 在根目录激活 `.venv` 后 `pip install <包名>`
3. **data-agent 无法调用 finance-mcp**: 确保 finance-mcp 先启动，且使用根虚拟环境
