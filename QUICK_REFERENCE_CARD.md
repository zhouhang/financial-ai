## 第3步：规则配置

### 你可以做什么

```
🌐 全局规则  - 容差、过滤、分组
📁 业务规则  - 仅业务文件(文件1)
📁 财务规则  - 仅财务文件(文件2)
🔀 混合规则  - 两个文件分别配置
```

### 关键原理：智能识别

系统会自动识别字段属于哪个文件：
- product_price 在业务文件 → 只改业务文件
- 发生- 在财务文件 → 只改财务文件

**您不用担心规则应用到错误的文件！**

### 快速命令

| 类型 | 输入示例 |
|------|---------|
| 容差 | "金额容差0.01元" |
| 业务转换 | "业务文件的revenue_amount除以100" |
| 财务转换 | "财务文件的发生-除以100" |
| 两个都转换 | "两个文件的订单号都去除空格" |
| 删除规则 | "删除金额容差" |

### 配置列表标记说明

```
🌐 全局配置    → 同时适用于两个文件
📁 业务文件    → 仅业务数据(文件1)
📁 财务文件    → 仅财务数据(文件2)
📁 业務+財務   → 为两个文件分别配置了
```

### 确认或继续添加

```
✅ 确认完成：输入"确认"  → 进入验证预览
➕ 继续添加：输入新规则  → 添加更多配置
❌ 删除规则：输入删除命令 → 移除已有规则
```

```
用户输入：

```

### 场景2：只改财务端

```
用户输入：
"财务文件的发生-除以100"

系统标记：
📁 财务文件(文件2)
```

### 场景3：两个都改（但字段不同）

```
用户输入①：
"业务文件的revenue_amount除以100"
→ 📁 业务文件

用户输入②：
"财务文件的income_money除以100"
→ 📁 财务文件
```

### 场景4：全局规则 + 文件规则

```
用户输入①：
"金额容差0.01元"
→ 🌐 全局配置

用户输入②：
"业务文件的product_price除以100"
→ 📁 业务文件

用户输入③：
"确认"
→ 进入验证预览
```

## ⚠️ 常见错误 vs 正确做法

| ❌ 错误 | ✅ 正确 | 为什么 |
|--------|--------|-------|
| "改金额" | "业务文件的revenue_amount除以100" | 需要具体字段名 |
| "两个都改金额" | 分别输入两次（字段名不同） | 不同文件字段名不同 |
| "product_price/100" | "product_price除以100" | 自然语言更清楚 |
| 修改不存在的字段 | 确保字段在映射中 | 系统无法处理虚拟字段 |

## 完整工作流示例

### 3分钟快速对账配置

```
1. 系统：展示初始字段映射
   
2. 用户: "确认"
   (字段映射正确)
   → 进入规则配置

3. 系统：要求配置规则
   
4. 用户: "金额容差0.01元"
   系统: ✅ 🌐 全局配置
   
5. 用户: "业务文件的amount除以100"
   系统: ✅ 📁 业务文件
   
6. 用户: "确认"
   → 进入验证预览

7. 系统：显示处理结果
   ✓ 业务数据：amount已/100
   ✓ 财务数据：amount未变
   
8. 用户: "确认"
   → 进入规则保存和对账执行
```

```

```

**原因**：字段名不在映射中
**解决**：
```
回到第2步，先在字段映射中添加该字段
```

## 关键概念速查

| 术语 | 说明 | 例子 |
|------|------|------|
| 数据源 | 业务或财务文件 | business/finance |
| 字段角色 | 标准字段类型 | order_id, amount, date, status |

## 常用规则模板

### 转换规则

```
金额 / 100        → "revenue_amount除以100"
金额 * 100        → "amount乘以100"
去除空格          → "订单号去除空格"
去除前导0         → "订单号去掉前导0"
去除前缀          → "订单号去掉前缀'ORD-'"
截取前N位        → "订单号只保留前21位"
日期格式          → "日期改为YYYY-MM-DD格式"
```

### 全局规则

```
金额容差          → "金额容差 X 元"
订单过滤          → "只对账订单号以'104'开头的"
分组累加          → "相同订单号做金额累加"
```
