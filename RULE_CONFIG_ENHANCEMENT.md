# 规则配置增强 - 两个文件独立处理指南

## 概述

规则配置阶段（第3步）已增强，现在完整支持：
- ✅ **全局规则**（同时对两个文件）
- ✅ **业务文件专用规则**（仅文件1）
- ✅ **财务文件专用规则**（仅文件2）
- ✅ **混合规则**（两个文件不同的转换）

## 核心改进

### 1. 智能数据源识别

系统现在能够根据**字段名**自动识别字段属于哪个文件：

```
业务数据字段         财务数据字段
├─ order_id         ├─ order_id
├─ amount           ├─ amount
├─ date             ├─ date
├─ status           ├─ status
├─ product_price    ├─ 发生-
├─ ...              ├─ ...
```

当用户说"product_price除以100"时：
- ✅ 系统识别 product_price 在业务数据中
- ✅ 只在 business 配置转换规则
- ✅ 财务数据的金额字段不受影响

### 2. 规则配置类型

```
规则配置
├─ 全局规则 🌐
│  ├─ 金额容差
│  ├─ 过滤规则
│  └─ 字段分组
├─ 业务文件规则 📁 文件1
│  ├─ 字段转换
│  └─ 字段清理
└─ 财务文件规则 📁 文件2
   ├─ 字段转换
   └─ 字段清理
```

## 使用场景和示例

### 场景1：只修改业务端的金额

**背景**：
- 业务文件中，金额存储为分（需要除以100）
- 财务文件中，金额已经是元（无需转换）

**您应该输入**：
```
业务文件的product_price除以100
或
文件1的金额字段需要除以100
```

**系统会执行**：
```
✅ 已添加配置：业务端转换：product_price除以100

当前配置：
  1. 📁 业务文件(文件1) 业务端转换：product_price除以100
```

**实际生成的规则**：
```json
{
  "data_cleaning_rules": {
    "business": {
      "field_transforms": [
        {
          "field": "amount",
          "transform": "float(row.get('product_price', 0)) / 100",
          "description": "product_price 除以100转换为元"
        }
      ]
    }
  }
}
```

### 场景2：两个文件都需要转换（但列名不同）

**背景**：
- 业务文件：revenue_amount（需要除以100）
- 财务文件：income_amount（需要除以100）

**您应该输入**：
```
业务文件的revenue_amount除以100，财务文件的income_amount也除以100
或
文件1和文件2的金额字段都需要除以100
```

**系统会执行**：
```
✅ 已添加配置：业务端转换：revenue_amount除以100
✅ 已添加配置：财务端转换：income_amount除以100

当前配置：
  1. 📁 业务文件(文件1) 业务端转换：revenue_amount除以100
  2. 📁 财务文件(文件2) 财务端转换：income_amount除以100
```

**实际生成的规则**：
两个分别的配置：
```json
{
  "data_cleaning_rules": {
    "business": {
      "field_transforms": [{
        "field": "amount",
        "transform": "float(row.get('revenue_amount', 0)) / 100"
      }]
    }
  }
}

{
  "data_cleaning_rules": {
    "finance": {
      "field_transforms": [{
        "field": "amount", 
        "transform": "float(row.get('income_amount', 0)) / 100"
      }]
    }
  }
}
```

## 提示和实现细节

### UI反馈增强

现在配置列表会显示每个规则的应用范围：

```
🌐 全局配置 - 适用于两个文件或全局设置
📁 业务文件(文件1) - 仅在业务数据中应用
📁 财务文件(文件2) - 仅在财务数据中应用
📁 业务+财务文件 - 为两个文件分别配置了规则
```

### LLM Prompt 增强内容

新prompt包含：
- ✅ 清晰的字段映射关系表（两个文件的所有字段）
- ✅ 数据源识别规则（根据字段名判断）
- ✅ 关键字匹配规则（"文件1"、"业务"等）
- ✅ 转换规则隔离警告（不要为两个文件都配置除非明确指定）
- ✅ 4个实时示例（展示不同场景）

### 字段名识别逻辑

```python
def identify_data_source(field_name: str, field_mappings: dict) -> str:
    """识别字段所属的数据源"""
    # 1. 检查是否在业务数据字段中
    if field_name in business_fields:
        return "business"  # 只在 business 中配置
    
    # 2. 检查是否在财务数据字段中
    if field_name in finance_fields:
        return "finance"   # 只在 finance 中配置
    
    # 3. 用户明确指定（"文件1"、"业务文件"等）
    if user_mentions_file1 or user_mentions_business:
        return "business"
    
    if user_mentions_file2 or user_mentions_finance:
        return "finance"
    
    # 4. 默认：两个文件都配置（全局规则）
    return "both"
```

## 关键规则和约束

### ✅ 支持的操作

| 操作 | 适用范围 | 说明 |
|------|--------|------|
| 添加转换规则 | 单文件 | 业务或财务文件单独配置 |
| 添加全局规则 | 两个文件 | 容差、过滤、分组等 |
| 混合配置 | 灵活 | 一次添加业务+财务的不同规则 |
| 删除规则 | 任意 | 删除任何已添加的配置 |

### ❌ 应避免的做法

```
❌ 错误：用户说"金额除以100"系统为两个文件都配置
✅ 正确：根据field mapping识别，只在对应文件配置

❌ 错误：用户说"product_price除以100"系统也改"发生-"
✅ 正确：product_price在业务文件，只改业务文件

❌ 错误：互相覆盖规则
✅ 正确：新规则与已有规则合并
```

## 与字段映射的关联

### 工作流程

```
第2步：字段映射
  → 确认每个字段属于哪个文件
  → 例如：product_price 在业务文件，发生- 在财务文件

        ↓

第3步：规则配置
  → 系统有了字段映射信息
  → 用户说"product_price除以100"
  → LLM查看字段映射，识别此字段在业务文件
  → 只在business配置规则 ✅
```

### 避免的问题

这个增强直接解决了您之前报告的bug：

**问题**：
```
用户输入："product_price要除以100"
旧系统：为business配置 ✓，也为finance配置 ✗
结果：财务文件的"发生-"也被错误地除以100
```

**现在**：
```
用户输入："product_price要除以100"
新系统：
  1. 查看字段映射
  2. product_price 在 business 中
  3. 发生- 在 finance 中
  4. 只在 business 配置规则 ✓
结果：仅业务文件转换，财务文件不影响 ✓
```

## 故障排查

### Q: 系统说"无法自动解析"

可能的原因：
- 指令不够清晰（例如："改一下"）
- 没有提及具体字段名
- 字段名不在映射中

**解决**：
```
❌ 不清楚: "业务文件的金额转换"
✅ 清楚: "业务文件的revenue_amount除以100"
```

### Q: 规则应用到了两个文件

检查：
1. 是否明确说了"两个文件都"？
2. 字段名是否在映射中？
3. 用户指令中是否有歧义？

**确认方式**：
- 查看规则配置列表的标记（🌐 vs 📁）
- 清楚地标注了应用范围

### Q: 如何验证规则只应用于一个文件

方法：
1. 查看UI中的标记
   - 📁 业务文件(文件1) - 只在业务数据
   - 📁 财务文件(文件2) - 只在财务数据
2. 进入验证预览（第4步）看结果
3. 查看日志中的JSON片段

## 最佳实践

### ✅ 推荐做法

1. **为每个文件分别输入规则**
   ```
   第1次："业务文件的revenue_amount除以100"
   第2次："金额容差0.01"
   ```

2. **明确指定文件**
   ```
   "文件1的订单号..."
   "业务数据的金额..."
   "财务文件的..."
   ```

3. **确认规则列表**
   - 每条规则都有数据源标记
   - 确认应用范围如预期

4. **进入验证预览查看结果**
   - 两个文件的数据是否正确处理
   - 是否需要调整

### ❌ 需要避免

1. **模糊的指令**
   ```
   "改一下金额"
   "处理订单号"
   ```

2. **对同名字段的假设**
   ```
   假设：两个文件都有"amount"
   现实：可能一个叫"revenue"，一个叫"income"
   ```

3. **试图用一条规则处理不同的字段**
   ```
   "两个文件的金额都除以100"（如果字段名不同）
   → 应该分别输入，名字要具体
   ```
