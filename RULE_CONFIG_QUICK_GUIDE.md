# 规则配置快速参考指南

## 现在您可以做什么

在**第3步：配置对账规则参数**阶段，您可以为两个文件独立配置规则。

---

## 快速示例

### 示例1：只修改业务文件的金额

```
用户: 业务文件的product_price除以100
结果: ✅ 📁 业务文件(文件1) 业务端转换：product_price除以100
```

### 示例2：两个文件各自转换（不同字段）

```
用户（第1次）: 业务文件的revenue_amount除以100
结果: ✅ 📁 业务文件(文件1) revenue_amount除以100

用户（第2次）: 财务文件的income_money除以100  
结果: ✅ 📁 财务文件(文件2) income_money除以100
```

### 示例3：全局容差 + 文件特定规则

```
用户（第1次）: 金额容差0.01元
结果: ✅ 🌐 全局配置 金额容差：0.01元

用户（第2次）: 文件1的revenue_amount除以100
结果: ✅ 📁 业务文件(文件1) revenue_amount除以100
```

### 示例4：两个文件都做相同转换

```
用户: 两个文件的订单号都需要去除空格
结果: ✅ 📁 业务+财务文件 订单号字段去除空格
```

---

## UI标记说明

```
🌐 全局配置    → 全局规则（容差、过滤等）或两个文件共用
📁 业务文件(文件1)  → 仅在业务数据应用
📁 财务文件(文件2)  → 仅在财务数据应用
📁 业务+财务文件  → 为两个文件分别配置了相同类型的规则
```

## 关键点

### ✅ 系统会自动识别

当您说"product_price除以100"时：
- 系统查看字段映射
- product_price 在业务文件中
- **自动只在业务文件配置规则** ✓
- 财务文件不受影响 ✓

### ⚠️ 确保清晰

```
❌ 模糊: "改金额"
✅ 清楚: "业务文件的revenue_amount除以100"

❌ 模糊: "去处理订单号"
✅ 清楚: "两个文件的订单号都去除空格"
```

### 🔍 验证方式

每次添加配置后，查看列表：
- 标记是否正确（📁 vs 🌐）
- 是否应用于预期的文件

## 常见问题快速回答

**Q: 为什么我的规则没有应用到财务文件？**

可能原因：
- 字段名不在财务文件中
- 系统自动识别为仅业务端规则（检查UI标记）

解决：如果需要两个文件都有规则，分别输入两次

---

**Q: 怎样验证规则只应用于一个文件？**

方法：
1. 查看UI中的标记
   - 📁 业务文件(文件1) → 仅业务
   - 📁 财务文件(文件2) → 仅财务
2. 进入验证预览看结果

---

**Q: 可以同时删除多条规则吗？**

可以，按顺序：
```
用户: 删除金额容差和订单号处理
```

---

**Q: 规则的顺序重要吗？**

不重要。系统会按照逻辑顺序应用规则。
