# ✅ 保存规则时去掉不需要的中间信息 - 修复完成

## 问题描述

在保存规则时，出现了两条不相关的消息：
1. **"xifu"** ← 不需要的翻译中间输出
2. **"规则 **西福** 已保存！..."** ← 需要的确认消息

这是因为 `save_rule_node` 中的 LLM 调用（用于翻译规则名称）的输出被流式捕获并发送给了前端。

---

## 🔍 根本原因

### 问题流程

```
save_rule_node 执行流程：
  1. 用户输入规则名称：西福
  2. 调用 _translate_rule_name_to_english(rule_name_cn="西福")
  3. LLM 翻译输出：xifu
  4. ❌ 这个中间输出被 astream_events 捕获
  5. WebSocket 流式发送给前端
  6. 用户看到两条消息
```

### 代码位置

在 [server.py#L340-L350](finance-agents/data-agent/app/server.py#L340-L350)：

```python
# 旧代码：只过滤了部分节点
if node_name not in ["file_analysis", "field_mapping", "rule_config", "validation_preview"]:
    # 流式发送
    # ❌ save_rule 的输出也被发送了
```

---

## 🔧 修复方案

### 修改内容

将 `save_rule` 添加到过滤列表中，使其不会流式发送内部的 LLM 输出：

```python
# 新代码：添加 save_rule 到过滤列表
if node_name not in ["file_analysis", "field_mapping", "rule_config", "validation_preview", "save_rule"]:
    # ✅ save_rule 的中间输出不再被发送
    # 只有最终的确认消息通过 on_chain_end 被发送
```

### 修改文件

| 文件 | 行号 | 修改 |
|------|------|------|
| [server.py](finance-agents/data-agent/app/server.py) | 350 | 添加 `"save_rule"` 到过滤列表 |

### 修改原理

```
save_rule_node 节点的消息处理：

┌─────────────────────────────────────┐
│ save_rule_node 执行                 │
├─────────────────────────────────────┤
│ ① LLM 翻译名称：_translate_rule...() │
│   输出：xifu                        │
│   ❌ 被 on_chat_model_stream 捕获   │
│   ✅ 修复后被过滤，不发送            │
├─────────────────────────────────────┤
│ ② 返回确认消息                      │
│   消息：规则 **西福** 已保存！      │
│   ✅ 通过 on_chain_end 正常发送      │
└─────────────────────────────────────┘
```

## 📝 过滤列表完整说明

现在过滤列表包含以下节点，它们的 LLM 输出不会被流式发送：

| 节点 | 说明 | 原因 |
|------|------|------|
| **file_analysis** | 文件分析 | 生成内部的字段分析，用户不需要看 |
| **field_mapping** | 字段映射 | 生成内部的字段映射，用户不需要看 |
| **rule_config** | 规则配置 | 配置项处理通过 interrupt 显示，不需要流式 |
| **validation_preview** | 验证预览 | 生成规则 schema，最后显示完整预览结果 |
| **save_rule** | 规则保存 | ✨ **新增** 中间的名称翻译输出（xifu）不需要显示 |

### 不被过滤的节点

| 节点 | 说明 |
|------|------|
| **router** | 路由/对话理解，面向用户的输出 |
| **其他** | 任何直接返回给用户的消息节点 |
