## 一、业务概述

### 4.2 计算规则

```python
未收金额 = 应收金额 - 已收金额
账龄天数 = (当前日期 - 业务日期).days
逾期天数 = 账龄天数 - 信用期限
if 逾期天数 < 0:
    逾期天数 = 0
```

## 五、分析规则

### 5.1 逾期判定

```python
if 逾期天数 > 0:
    状态 = "逾期"
elif 账龄天数 > 180:
    状态 = "关注"
else:
    状态 = "正常"
```

### 5.2 坏账风险判定

**高风险条件**（满足任一即可）:

| 条件 | 判定标准 |
|------|----------|
| 逾期天数 | 逾期 > 365 天 |
| 账龄 | 账龄 > 2 年 |
| 客户状态 | 客户已注销/失联 |
| 诉讼情况 | 涉及诉讼/被执行 |

### 5.3 客户集中度分析

```python
客户集中度 = 该客户应收金额 / 应收款项总额×100%
```

**风险阈值**:
- 单一客户占比 > 30%: 集中度高
- 前五名占比 > 70%: 集中度高

### 5.4 周转率计算

```python
应收账款周转率 = 赊销收入净额 / 平均应收账款余额
应收账款周转天数 = 365 / 周转率
平均应收账款余额 = (期初应收 + 期末应收) / 2
```

## 六、汇总统计规则

### 6.1 按账龄汇总

```python
按账龄分类分组：
- 各账龄客户数 = count(distinct 客户)
- 各账龄金额 = sum(未收金额)
- 各账龄占比 = 各账龄金额 / 总应收金额
```

### 6.2 按客户汇总

```python
按客户分组：
- 客户应收总额 = sum(应收金额)
- 客户已收总额 = sum(已收金额)
- 客户未收总额 = sum(未收金额)
- 逾期金额 = 未收金额.where(逾期天数 > 0).sum()
```

### 6.3 账龄结构分析

| 账龄分类 | 客户数 | 金额 | 占比 | 逾期金额 |
|----------|--------|------|------|----------|
| 1 月以内 | X | A | A% | - |
| 1-2 月 | Y | B | B% | - |
| 2-3 月 | Z | C | C% | - |
| 3-6 月 | W | D | D% | D_overdue |
| 6 月 -1 年 | V | E | E% | E_overdue |
| 1 年以上 | U | F | F% | F_overdue |

## 八、校验规则

1. **金额平衡**: ∑应收金额 - ∑已收金额 = ∑未收金额
2. **账龄校验**: 账龄 = 当前日期 - 业务日期
3. **分类完整**: 各账龄分类金额合计 = 总应收金额

## 十、注意事项

### 10.1 特殊情况处理

1. **部分回款**: 按先发生先收回原则核销
2. **票据结算**: 应收票据视同已收
3. **债务重组**: 单独标记重组债权
4. **关联方往来**: 与外部客户分开统计

### 10.2 账龄计算方法

**方法 1：先进先出法**
- 按业务日期顺序核销
- 先发生的先收回

**方法 2：个别认定法**
- 按单据逐笔核销
- 适用于能对应具体回款的场景

### 10.3 坏账准备计提参考

| 账龄 | 建议计提比例 |
|------|--------------|
| 1 月以内 | 0% |
| 1-3 月 | 5% |
| 3-6 月 | 20% |
| 6 月 -1 年 | 50% |
| 1 年以上 | 100% |
