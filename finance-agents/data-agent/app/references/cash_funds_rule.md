# 货币资金明细表数据整理规则

**规则编码**: `RULE_CASH_FUNDS_001`
**版本**: 1.0
**业务类型**: `cash_funds`
**最后更新**: 2026-02-26

---

## 一、业务概述

本规则用于从企业财务数据中整理生成**货币资金明细表**，包括库存现金、银行存款和其他货币资金的期初余额、本期发生额、期末余额，并与银行对账单金额进行比对分析。

---

## 二、输出数据结构

### 2.1 货币资金明细表字段

| 序号 | 字段名 | 说明 | 数据来源 |
|------|--------|------|----------|
| 1 | 序号 | 自动编号 | 生成 |
| 2 | 科目名称 | 库存现金/银行存款/其他货币资金 | 科目余额表 |
| 3 | 核算项目 | 银行账户明细 | 科目余额表 |
| 4 | 期初金额 | 期初借方 - 期初贷方 | 计算 |
| 5 | 本期借方 | 本期发生额借方 | 科目余额表 |
| 6 | 本期贷方 | 本期发生额贷方 | 科目余额表 |
| 7 | 期末金额 | 期末借方 - 期末贷方 | 计算 |
| 8 | **银行对账单金额** | 银行方提供的期末余额 | 银行对账单 |
| 9 | 差异 | 期末金额 - 银行对账单金额 | 计算 |
| 10 | 账户性质 | 现金/银行存款/其他货币资金 | 推导 |
| 11 | 备注 | 特殊说明 | 生成 |

## 三、数据源识别规则

### 3.1 科目余额表识别

**文件名匹配规则**:
- 文件名包含：`科目余额表`

**文件内容特征**（满足以下任一即可）:
- 包含列：`科目代码 `、`科目名称`、` 核算项目`、`年初余额`、` 期初余额`、` 本期发生额`、` 期末余额`

### 3.2 银行对账单识别

**文件名匹配规则**:
- 文件名包含：`银行对账单` 或 `银行自有对账单`

**文件内容特征**（满足以下任一即可）:
- 包含列：`账号 `、`账号名称`、` 币种`、`交易日`、` 借方金额`、` 贷方金额`、` 余额`
- 包含列：`发生时间 `、`账户名`、` 账户类别`、` 账号`、`发生 +`、` 发生 -`、` 流水号`

## 四、数据提取规则

### 4.1 科目余额表数据提取

**过滤条件**:
```
科目名称 IN ('库存现金%', '银行存款%', '其他货币资金%')
```

**科目名称匹配模式**:
- `库存现金` - 精确匹配或前缀匹配
- `银行存款_*` - 以"银行存款_"开头
- `其他货币资金` - 精确匹配或前缀匹配

**列映射规则**:

| 标准字段 | 可能的列名（兼容多种命名） |
|----------|---------------------------|
| 科目名称 | `科目名称`、`会计科目`、`科目` |
| 核算项目 | `核算项目`、`项目核算`、`辅助核算` |
| 期初借方 | `期初余额借方 `、` 年初余额借方 `、` 期初借方` |
| 期初贷方 | `期初余额贷方 `、` 年初余额贷方 `、` 期初贷方` |
| 本期借方 | `本期发生额借方 `、` 本期借方 `、` 本年累计借方` |
| 本期贷方 | `本期发生额贷方 `、` 本期贷方 `、` 本年累计贷方` |
| 期末借方 | `期末余额借方 `、` 期末借方` |
| 期末贷方 | `期末余额贷方 `、` 期末贷方` |

**计算规则**:
```python
期初金额 = 期初借方 - 期初贷方
本期借方 = 直接读取
本期贷方 = 直接读取
期末金额 = 期末借方 - 期末贷方
```

## 五、银行对账单金额取数规则

### 5.1 重要提示：数据源区分

**⚠️ 严禁从银行日记账中取数！**

| 单据类型 | 英文名称 | 记录方 | 是否可用 |
|----------|----------|--------|----------|
| 银行对账单 | Bank Statement | 银行方 | ✅ 可用 |
| 银行自有对账单 | Bank Internal Statement | 银行方 | ✅ 可用 |
| 银行日记账 | Bank Journal | 企业方 | ❌ **严禁使用** |

### 5.2 银行信息提取

**核算项目格式**: `银行账户：{银行名称}{账号}`

**提取规则**:
```python
银行名称 = "银行账户："之后、数字之前的部分
账号后 5 位 = 最后 5 位数字
```

**示例**:
| 核算项目 | 银行名称 | 账号后 5 位 |
|----------|----------|------------|
| 银行账户：中国银行 72515 | 中国银行 | 72515 |
| 银行账户：招商银行 10001 | 招商银行 | 10001 |
| 银行账户：支付宝 flfq20250902@163.com | 支付宝 | (无，跳过) |

### 5.3 Sheet 选择规则

**Sheet 名称匹配优先级**:

1. **第一优先级**: Sheet 名称包含 `银行名称 + 对账单`
   - 如：匹配"中国银行"时，优先查找"中国银行对账单"

2. **第二优先级**: Sheet 名称包含 `银行对账单` 且账户名称包含银行名称

3. **第三优先级**: Sheet 名称包含 `自有对账单` 且账户名称包含银行名称

4. **⚠️ 禁止**: 严禁使用名称中包含 `日记账` 的 Sheet

### 5.4 余额读取规则

**取数规则**:
- 从匹配到的银行对账单中，找到所有账号后 5 位匹配的记录
- 取**最后一笔交易**的"余额"列值
- 如果未找到匹配记录，银行对账单金额为 0

**余额列的可能名称**:
- `余额`、` 账户余额`、` 当前余额`、` 流水余额`

**数据格式处理**:
```python
# 移除千位分隔符
余额 = 余额.replace(',', '')
# 提取数值（如果包含借/贷标识）
余额 = re.extract(r'[\d.]+', 余额)
```

## 六、差异计算规则

```python
差异 = 期末金额 - 银行对账单金额
差异 = round(差异，2)  # 四舍五入保留 2 位小数
```

## 八、特殊情况处理

### 8.1 第三方支付账户

对于支付宝、微信支付等第三方支付账户：
- 核算项目通常不包含标准银行名称和账号
- 银行对账单金额 = 0
- 差异 = 期末金额

### 8.2 未匹配到银行对账单

如果核算项目中没有明确的银行名称或账号：
- 银行对账单金额 = 0
- 在备注中标注"未匹配到银行对账单"

### 8.3 同一银行多账号

如果同一银行有多个账号：
- 必须通过账号后 5 位精确匹配到具体账户
- 每个账号独立计算差异

## 十、处理流程

```
1. 识别并读取科目余额表
   ↓
2. 过滤货币资金相关科目
   ↓
3. 计算期初、本期、期末金额
   ↓
4. 识别并读取银行对账单
   ↓
5. 提取银行信息（名称 + 账号后 5 位）
   ↓
6. 匹配银行对账单 Sheet
   ↓
7. 读取最后一笔交易余额
   ↓
8. 计算差异
   ↓
9. 生成货币资金明细表
```

---

## 十一、校验规则

1. **期初平衡校验**: 期初借方合计 - 期初贷方合计 = 期初总余额
2. **期末平衡校验**: 期末借方合计 - 期末贷方合计 = 期末总余额
3. **差异率计算**: 差异率 = |差异 | / |期末金额 | * 100%
4. **异常阈值**: 差异率 > 5% 时标注"需关注"
