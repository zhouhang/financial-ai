# 货币资金明细表数据整理描述
- name: 货币资金数据整理skill
- desc: 从指定文件中读取对应的贷币资金核对数据所需的数据，生成新的货币资金明细表 (markdown 格式)

## 货币资金明细表包括有列
- 序号
- 科目名称
- 核算项目
- 期初金额
- 本期借方
- 本期贷方
- 期末金额
- **银行对账单金额**
- 差异
- 账户性质
- 备注

# 货币资金明细表 (markdown 格式) 中列的取数规则

1. 首先从"科目余额表"文件中过滤取科目名称含有:'库存现金'、'银行存款'、'其他货币资金'等的数据记录，

2. 科目名称、核算项目从第 1 步过滤后的记录中读取数据；

3. **重要**：科目余额表中的余额列通常分为"借方"和"贷方"两列，列名可能的形式包括：
   - 期初余额：`期初余额借方`/`期初余额贷方` 或 `年初余额借方`/`年初余额贷方`
   - 本期发生额：`本期发生额借方`/`本期发生额贷方` 或 `本期借方`/`本期贷方` 或 `本年累计借方`/`本年累计贷方`
   - 期末余额：`期末余额借方`/`期末余额贷方`

   取数规则如下：
   - 期初金额 = 期初余额借方 - 期初余额贷方（或年初余额借方 - 年初余额贷方）
   - 本期借方 = 直接读取本期发生额借方（或本期借方、本年累计借方）
   - 本期贷方 = 直接读取本期发生额贷方（或本期贷方、本年累计贷方）
   - 期末金额 = 期末余额借方 - 期末余额贷方

4. 循环读取明细表数据，如果科目为"银行存款"相关的记录，则**银行对账单金额**列从**银行对账单表**中读取**期末余额**（取该账户最后一笔交易记录的"余额"列）；注意：具体从哪个银行的对账单中读取账单金额数据需要根据核算项目中的银行名称和账号后 5 位来判断和过滤；

5. "差异"列的值为：期末金额 - 银行对账单金额相减后，四舍五入取 2 位小数。

## 银行对账单金额取数详细规则

### ⚠️ 重要提示：数据源区分

**银行对账单** vs **银行日记账** - 这是两种不同的单据，必须严格区分：

| 单据类型 | 英文名称 | 记录方 | 用途 | 是否可用于取数 |
|----------|----------|--------|------|---------------|
| **银行对账单** | Bank Statement | 银行方 | 银行出具的企业账户交易明细 | ✅ **可以** |
| **银行自有对账单** | Bank Internal Statement | 银行方 | 银行系统内部的企业账户交易记录 | ✅ **可以** |
| **银行日记账** | Bank Journal | 企业方 | 企业自己记录的银行存款收支明细 | ❌ **严禁使用** |

**⚠️ 严禁从银行日记账中读取银行对账单金额！**

银行日记账是企业方根据自己的记账凭证记录的，不是银行方的官方记录，不能用于对账。

### 步骤 1: 从核算项目中提取银行信息

核算项目的格式通常为：`银行账户：{银行名称}{账号}`

**提取规则**:
- 银行名称 = "银行账户："之后、数字之前的部分
- 账号后 5 位 = 最后 5 位数字

**示例**:
| 核算项目 | 银行名称 | 账号后 5 位 |
|----------|----------|------------|
| 银行账户：中国银行 72515 | 中国银行 | 72515 |
| 银行账户：招商银行 10001 | 招商银行 | 10001 |
| 银行账户：兴业银行 17420 | 兴业银行 | 17420 |
| 银行账户：支付宝 flfq20250902@163.com | 支付宝 | (无，跳过匹配) |

### 步骤 2: 识别并选择正确的银行对账单 Sheet

**⚠️ Sheet 名称识别规则**:

| Sheet 名称包含 | 单据类型 | 是否可用 | 说明 |
|---------------|----------|----------|------|
| `银行对账单` | 银行对账单 | ✅ **可用** | 如"1、福禄控股银行对账单" |
| `自有对账单` | 银行自有对账单 | ✅ **可用** | 如"2、银行自有对账单" |
| `日记账` | 银行日记账 | ❌ **严禁使用** | 如"中国银行日记账"、"招商银行日记账" |
| `银行日记账` | 银行日记账 | ❌ **严禁使用** | 企业方记录，非银行方记录 |

**匹配优先级**:

1. **第一优先级**: Sheet 名称包含"`{银行名称}+ 对账单`"的 sheet
   - 示例：匹配"中国银行"时，优先查找"中国银行对账单"

2. **第二优先级**: Sheet 名称包含"`银行对账单`"且账户名称包含银行名称的 sheet
   - 示例："1、福禄控股银行对账单"，检查其中的"账户名"、"账户类别"列是否包含"中国银行"

3. **第三优先级**: Sheet 名称包含"`自有对账单`"且账户名称包含银行名称的 sheet
   - 示例："2、银行自有对账单"，检查其中的"账号名称"列是否包含"中国银行"

4. **⚠️ 禁止**: 严禁使用名称中包含"`日记账`"的 sheet

**匹配示例**:

```
核算项目：银行账户：中国银行 72515
  -> 银行名称：中国银行
  -> 账号后 5 位：72515

正确的匹配过程:
  1. 查找 Sheet 名称包含"中国银行"且包含"对账单"的 sheet
     -> 未找到
  
  2. 查找 Sheet 名称包含"银行对账单"的 sheet，检查账户名称
     -> 找到"1、福禄控股银行对账单"
     -> 检查该 sheet 的"账户名"或"账户类别"列，查找包含"中国银行"的记录
     -> 进一步检查"账号"列，查找后 5 位为"72515"的记录
  
  3. 如果找到匹配记录，取最后一笔交易的"余额"列值

错误的匹配过程（严禁）:
  ❌ 查找 Sheet 名称包含"中国银行日记账"的 sheet
  ❌ 从"中国银行日记账"中读取数据
```

### 步骤 3: 读取银行对账单金额

**取数规则**:
- 从匹配到的银行对账单中，找到所有账号后 5 位匹配的记录
- 取**最后一笔交易**的"余额"列值作为银行对账单金额
- 如果未找到匹配记录，银行对账单金额为 0

**余额列的可能名称**:
- `余额`
- `账户余额`
- `当前余额`
- `流水余额`
- `流水余额`

**数据格式处理**:
- 如果余额列包含"借"/"贷"标识，需要提取数值部分
- 如果余额列包含逗号（千位分隔符），需要移除逗号

**示例**:
```
银行对账单数据:
| 账号 | 余额 |
|------|------|
| 72515 | 100,000.00 |
| 72515 | 150,000.00 |
| 72515 | 200,000.00 |  <- 最后一笔交易

银行对账单金额 = 200,000.00
```

## 科目余额表的查找和判断依据

- 文件名中包括：科目余额表即表示该文件为科目余额表;
- 文件标题包含有：科目代码、科目名称、公司、核算项目、年初余额、期初余额、本期发生额、本年累计、期末余额等即可以认定为科目余额表;

## 银行对账单表的查找和判断依据

- 文件名中包括：银行对账单即表示该文件为银行对账单表;
- 文件标题包含有：账号、账号名称、币种、交易日、交易时间、起息日、交易类型、借方金额、贷方金额、余额、摘要等即可以认定为银行对账单;
- 文件标题包含有：发生时间、账户名、账户类别、账号、发生 +、发生 -、流水号、对方账户名、对方账号、备注摘要等即可以认定为银行对账单;

## 注意事项

### ⚠️ 禁止事项（重要）

1. **严禁从银行日记账中读取银行对账单金额**
   - 银行日记账是企业方的记录，不是银行方的官方记录
   - 银行日记账的 Sheet 名称通常包含"日记账"字样
   - 即使没有银行对账单可用，也不能从银行日记账读取数据，银行对账单金额应为 0

2. **严禁使用未匹配的银行账户数据**
   - 如果核算项目中没有明确的银行名称或账号，银行对账单金额应为 0

### ✅ 正确做法

1. **优先使用银行对账单**
   - Sheet 名称包含"银行对账单"的 sheet
   - Sheet 名称包含"自有对账单"的 sheet

2. **正确匹配银行和账号**
   - 同时匹配银行名称和账号后 5 位，确保准确性
   - 如果同一银行有多个账号，必须通过账号后 5 位精确匹配到具体账户

3. **正确读取余额**
   - 取最后一笔交易的余额列值
   - 处理余额列中的"借"/"贷"标识和千位分隔符

### 其他注意事项

- 银行对账单中列：发生 + 为借方，发生 - 为贷方；
- 银行对账单金额应取该账户的**期末余额**（最后一笔交易的余额列），而非累计发生额；
- 科目余额表如果借方/贷方分列，需要计算净额（借方 - 贷方）；
- 如果核算项目中没有明确的银行名称或账号（如"支付宝"、"微信支付"等第三方支付），银行对账单金额为 0。
