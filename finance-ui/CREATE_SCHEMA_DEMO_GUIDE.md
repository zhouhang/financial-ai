1. 打开浏览器访问 http://localhost:5175/
2. 在聊天输入框输入：

**预期结果**:

### 1.2 填写基本信息（1分钟）

**操作**:
1. 点击"开始创建规则"按钮
2. 选择工作类型：**数据整理**
3. 输入规则名称：**销售数据整理**
4. 观察 type_key 自动生成：`xiao_shou_shu_ju_zheng_li`
5. 输入描述：**整理每月销售数据，计算含税金额**
6. 点击"下一步"

**预期结果**:
- 模态框全屏打开
- 表单验证正常
- Type key 实时生成
- 唯一性检查通过（绿色对勾）

**讲解要点**:
- 中文自动转拼音
- 实时唯一性验证（防抖 500ms）
- 表单验证友好提示

### 1.4 添加步骤（2分钟）

#### 步骤 1: 数据提取

**操作**:
1. 点击左侧"添加步骤"按钮
2. 输入步骤名称：**提取销售数据**
3. 选择步骤类型：**数据提取**
4. 选择源文件：**sales_data.xlsx**
5. 输入目标名称：**sales_data**
6. 点击"保存配置"

**预期结果**:
- 步骤添加到左侧列表
- 步骤显示为激活状态（蓝色边框）
- 显示"步骤已更新"提示

#### 步骤 2: 数据转换

**操作**:
1. 再次点击"添加步骤"
2. 输入步骤名称：**计算含税金额**
3. 选择步骤类型：**数据转换**
4. 选择操作类型：**计算**
5. 输入表达式：`amount * 1.13`
6. 点击"保存配置"

**预期结果**:
- 第二个步骤添加成功
- 步骤列表显示两个步骤

#### 步骤 3: 条件逻辑

**操作**:
1. 添加第三个步骤
2. 输入步骤名称：**标记高价值客户**
3. 选择步骤类型：**条件逻辑**
4. 配置条件：
   - 如果 [金额] **大于** `10000`
   - 则标记为 **VIP客户**
5. 点击"保存配置"

**预期结果**:
- 三个步骤全部显示
- 条件逻辑配置正确

**讲解要点**:
- 6 种步骤类型支持
- 动态表单配置
- 实时保存反馈

## 🎯 演示场景 5: 6 种步骤类型（3分钟）

### 5.1 Extract（数据提取）

**配置**:
- 源文件：选择文件
- 目标名称：提取后的数据集名称

**用途**: 从 Excel 提取数据

---

### 5.2 Transform（数据转换）

**配置**:
- 数据源：要转换的数据集
- 操作类型：映射/筛选/计算
- 表达式：转换规则

**用途**: 对数据进行计算或转换

---

### 5.3 Validate（数据验证）

**配置**:
- 数据源：要验证的数据集
- 验证规则：必填/唯一/范围/格式

**用途**: 验证数据是否符合规则

---

### 5.4 Conditional（条件逻辑）

**配置**:
- 条件：列名、运算符、比较值
- Then 动作：条件为真时的操作
- Else 动作：条件为假时的操作

**用途**: 根据条件执行不同操作

---

### 5.5 Merge（数据合并）

**配置**:
- 合并类型：内连接/左连接/右连接/外连接
- 连接键：用于匹配的字段
- 目标名称：合并后的数据集名称

**用途**: 合并多个数据源

---

### 5.6 Output（输出配置）

**配置**:
- 数据源：要输出的数据集
- 输出列：选择要包含的列
- 输出格式：Excel/CSV/JSON

**用途**: 定义最终输出格式
