# Create Schema Canvas - 可视化数据处理规则创建工具

> 通过直观的画布界面，零代码创建数据整理和对账规则

## 🎯 功能概述

Create Schema Canvas 是一个集成在 Finance AI 助手中的可视化规则配置工具，让用户通过简单的拖拽和表单操作，即可创建复杂的数据处理规则。

### 核心特性

- ✅ **两步向导**: 简化的创建流程，从元数据到配置一气呵成
- ✅ **3列布局**: 步骤列表 | Excel预览 | 配置面板，一目了然
- ✅ **6种步骤类型**: 提取、转换、验证、条件、合并、输出
- ✅ **实时预览**: 上传Excel文件即时预览，支持多文件多Sheet
- ✅ **撤销重做**: 最多50步历史记录，随时回退
- ✅ **智能验证**: 实时检查配置正确性，提前发现问题
- ✅ **深色主题**: 护眼舒适的界面设计

## 📖 使用流程

### 步骤 1: 填写基本信息

```
┌─────────────────────────────────┐
│ 工作类型: ○ 数据整理 ○ 对账    │
│ 规则名称: [销售数据整理______]  │
│ 规则标识: [自动生成__________]  │
│ 描述说明: [可选______________]  │
│                                 │
│           [ 取消 ] [ 下一步 ]   │
└─────────────────────────────────┘
```

**功能说明**:
- 选择工作类型（数据整理或对账）
- 输入中文名称，系统自动生成英文标识
- 实时检查名称是否已存在
- 可选填写规则描述

### 步骤 2: 配置画布

```
┌────────────────────────────────────────────────────────┐
│ ← 返回  销售数据整理    [ 测试 ] [ 保存 ] [ 取消 ]    │
├──────────┬──────────────────────┬──────────────────────┤
│ 操作步骤 │   Excel 文件预览     │    步骤配置          │
│ ━━━━━━━ │   ━━━━━━━━━━━━━━━━ │    ━━━━━━━━━━━━━━ │
│          │                      │                      │
│ ↻ ↺      │  📁 sales.xlsx       │  步骤名称            │
│          │  ┌─────────────────┐ │  [提取销售数据___]   │
│ ▶ 步骤1  │  │ Sheet1  Sheet2  │ │                      │
│   步骤2  │  │ ┌─────────────┐ │ │  步骤类型            │
│   步骤3  │  │ │ 姓名 | 金额 │ │ │  [数据提取 ▼]        │
│          │  │ │ 张三 | 1000 │ │ │                      │
│ + 添加   │  │ └─────────────┘ │ │  [ 保存配置 ]        │
└──────────┴──────────────────────┴──────────────────────┘
```

**功能说明**:
- **左侧**: 管理所有处理步骤，支持添加、删除、撤销、重做
- **中间**: 预览上传的Excel文件，支持多文件和多Sheet切换
- **右侧**: 配置当前选中步骤的详细参数

## 🎨 步骤类型说明

### 1. 数据提取 (Extract)
从Excel文件中提取指定的数据

**配置项**:
- 源文件: 选择要提取的文件
- 目标名称: 提取后的数据集名称

**示例**: 从"销售明细.xlsx"中提取所有数据，命名为"sales_data"

---

### 2. 数据转换 (Transform)
对数据进行映射、筛选或计算

**配置项**:
- 数据源: 要转换的数据集
- 操作类型: 映射/筛选/计算
- 表达式: 转换规则

**示例**: 计算含税金额 `amount * 1.13`

---

### 3. 数据验证 (Validate)
验证数据是否符合规则

**配置项**:
- 数据源: 要验证的数据集
- 验证规则: 必填/唯一/范围/格式

**示例**: 验证金额字段必须大于0

---

### 4. 条件逻辑 (Conditional)
根据条件执行不同的操作

**配置项**:
- 条件: 列名、运算符、比较值
- Then动作: 条件为真时的操作
- Else动作: 条件为假时的操作（可选）

**示例**: 如果金额>10000，则标记为"VIP客户"

---

### 5. 数据合并 (Merge)
合并多个数据源

**配置项**:
- 合并类型: 内连接/左连接/右连接/外连接
- 连接键: 用于匹配的字段
- 目标名称: 合并后的数据集名称

**示例**: 将销售数据和客户信息按客户ID合并

---

### 6. 输出配置 (Output)
定义最终输出格式

**配置项**:
- 数据源: 要输出的数据集
- 输出列: 选择要包含的列
- 输出格式: Excel/CSV/JSON

**示例**: 输出为Excel文件，包含姓名、金额、日期三列

## 🔧 高级功能

### 撤销/重做

- **撤销**: 点击 ↻ 按钮或按 `Ctrl+Z`
- **重做**: 点击 ↺ 按钮或按 `Ctrl+Y`
- **历史记录**: 最多保存50步操作

### 测试功能

在保存前可以测试规则：
1. 点击顶部"测试"按钮
2. 系统验证配置并执行
3. 显示预期输出和执行时间
4. 如有错误，显示详细信息

### 实时验证

- **名称唯一性**: 输入名称时自动检查是否重复
- **配置验证**: 保存前自动验证所有步骤配置
- **文件验证**: 上传时检查文件格式和大小

## 📁 文件存储

### 目录结构

```
data_preparation/
├── schemas/
│   └── {user_id}/
│       └── {type_key}.json          # 规则配置文件
└── config/
    └── {user_id}/
        └── data_preparation_schemas.json  # 规则索引

reconciliation/
├── schemas/
│   └── {user_id}/
│       └── {type_key}.json
└── config/
    └── {user_id}/
        └── reconciliation_schemas.json
```

### Schema JSON 格式

```json
{
  "version": "1.0",
  "schema_type": "step_based",
  "metadata": {
    "project_name": "销售数据整理",
    "author": "username",
    "description": "整理每月销售数据"
  },
  "processing_steps": [
    {
      "step_name": "提取销售数据",
      "step_type": "extract",
      "config": {
        "source_file": "file-123",
        "target_name": "sales_data"
      },
      "order": 0
    }
  ]
}
```

## 🎯 使用场景

### 场景 1: 销售数据整理

**需求**: 每月整理销售数据，计算含税金额，标记VIP客户

**步骤**:
1. 提取销售数据
2. 计算含税金额 (金额 × 1.13)
3. 标记VIP客户 (金额 > 10000)
4. 输出为Excel

---

### 场景 2: 财务对账

**需求**: 对比银行流水和系统记录，找出差异

**步骤**:
1. 提取银行流水
2. 提取系统记录
3. 按交易号合并
4. 验证金额是否一致
5. 输出差异报告

---

### 场景 3: 客户数据清洗

**需求**: 清洗客户数据，去重并补充信息

**步骤**:
1. 提取客户数据
2. 验证必填字段
3. 去除重复记录
4. 补充客户等级
5. 输出清洗后数据

## 🐛 常见问题

### Q: 点击"开始创建规则"没反应？

**A**: 检查以下几点：
1. 浏览器控制台是否有错误
2. 后端服务是否正常运行
3. 刷新页面重试

### Q: 文件上传失败？

**A**: 确认：
1. 文件格式为 .xlsx 或 .xls
2. 文件大小不超过 100MB
3. 文件未损坏

### Q: 名称验证一直转圈？

**A**: 可能原因：
1. 网络连接问题
2. 后端服务未响应
3. 等待2-3秒后重试

### Q: 保存失败？

**A**: 检查：
1. 是否已添加处理步骤
2. 所有步骤配置是否完整
3. 查看错误提示信息
