# 对账异常摘要展示优化设计

## 背景

当前对账异常在异常列表、异常详情、公开查看页面中重复展示同一段长摘要。例如：

```text
仅 tb0131100248-店铺订单 存在（交易订单明细表 缺失）：订单编号=5118002676174023242
含税销售金额 ↔ 买家实付金额：tb0131100248-店铺订单 0.00
```

这段信息同时承担异常类型、缺失数据集、匹配字段、匹配值、对比字段和值，导致财务人员在列表中扫读成本高。列表应先回答“需要处理什么异常”，详情再承载证据和字段值。

## 目标

1. 异常列表只展示“差异类型 + 匹配字段 + 匹配值”的业务结论。
2. 缺失类异常必须明确“哪个数据集缺失哪个匹配字段值”。
3. 金额、字段差异、完整原始记录只放在详情中展示。
4. 展示文案使用真实数据集名称和业务字段名，不出现“左侧”“右侧”等技术方位词。
5. 不改变后端异常存储结构；前端基于现有结构化字段生成财务可读摘要。

## 非目标

1. 不重写对账执行逻辑。
2. 不调整异常生成和入库字段。
3. 不改变异常处理状态、责任人、催办流程。
4. 不改变导出的对账结果文件。

## 当前可用数据

前端公开异常页已经能读取以下结构：

1. `anomaly_type`：异常类型，例如 `source_only`、`target_only`、`matched_with_diff`。
2. `detail_json.join_key`：匹配字段和值。
3. `detail_json.compare_values`：差异字段及双方值。
4. `detail_json.raw_record`、`left_record`、`right_record` 等原始记录。
5. 对账方案中的数据集标签、输出字段标签、源字段标签。

因此第一版应优先在展示层生成短摘要，后端 `summary` 保留为兼容和兜底数据。

## 列表展示规则

### 缺失类异常

当异常类型为 `source_only` 或 `target_only` 时，列表摘要格式为：

```text
{缺失数据集名称}缺失{匹配字段名} {匹配值}
```

示例：

```text
交易订单明细表缺失订单编号 5118002676174023242
```

如果匹配字段不是订单编号，则动态使用对账方案中的匹配字段展示名：

```text
支付宝账单缺失商户订单号 202605280001
```

### 差异类异常

当异常类型为 `matched_with_diff` 或 `value_mismatch` 时，列表摘要格式为：

```text
{匹配字段名} {匹配值} {差异类型}
```

示例：

```text
订单编号 5118002676174023242 金额不一致
```

差异类型优先根据 `compare_values` 的字段语义推断：

1. 金额字段差异显示为 `金额不一致`。
2. 状态字段差异显示为 `状态不一致`。
3. 其他字段显示为 `{字段名}不一致`。
4. 多个字段同时不一致时显示 `多字段不一致`。

### 兜底规则

如果无法从 `join_key` 取得匹配字段和值：

1. 优先从原始 `summary` 中解析现有 `字段=值`。
2. 仍无法解析时显示原有格式化摘要。
3. 不显示空的“订单”“该订单”等含糊文案。

## 详情展示规则

详情页分块展示信息，不在标题区重复长摘要。

### 异常结论

显示与列表一致的业务结论，例如：

```text
交易订单明细表缺失订单编号 5118002676174023242
```

### 对账关键字段

使用真实数据集名称展示匹配字段和值：

```text
tb0131100248-店铺订单：订单编号 = 5118002676174023242
交易订单明细表：订单编号 = --
```

### 差异字段和值

列头使用真实数据集名称，不使用“左侧”“右侧”：

| 字段 | tb0131100248-店铺订单 | 交易订单明细表 | 差异值 |
| --- | --- | --- | --- |
| 含税销售金额 / 买家实付金额 | 0.00 | -- | -- |

### 原始记录

按源数据集分区展示各自所有字段：

1. 有原始记录的数据集展示该记录全部字段。
2. 缺失的数据集显示 `未匹配到原始记录`。
3. 字段标签使用方案中的业务字段名；无映射时使用原字段名。

## 影响范围

主要影响前端展示：

1. `finance-web/src/components/PublicReconRunExceptionsPage.tsx`
2. `finance-web/src/components/ReconWorkspace.tsx`
3. `finance-web/src/components/recon/ExceptionSummary.tsx`
4. 可新增一个小型展示 helper，例如 `finance-web/src/components/recon/exceptionBusinessSummary.ts`

`PublicReconRunExceptionsPage.tsx` 和 `ReconWorkspace.tsx` 已经较大，实现时应优先抽出 helper，避免继续把展示规则堆进大文件。

## 测试策略

1. 单元测试业务摘要生成 helper：
   - `source_only` 时显示目标数据集缺失匹配字段值。
   - `target_only` 时显示另一侧数据集缺失匹配字段值。
   - 金额差异显示 `{匹配字段名} {匹配值} 金额不一致`。
   - 匹配字段名从对账方案字段映射动态取得，不写死订单编号。
   - 无结构化字段时回退原摘要。
2. 组件测试：
   - 对账中心异常列表不再显示金额行和“匹配字段/对比字段”标签。
   - 公开异常页列表只显示短业务结论。
   - 详情页差异字段列头使用真实数据集名称。
   - 原始记录按数据集分区展示，缺失侧显示未匹配到原始记录。

## 验收标准

1. 示例异常在列表中显示为：

```text
交易订单明细表缺失订单编号 5118002676174023242
```

2. 列表中不再显示：
   - `含税销售金额 ↔ 买家实付金额`
   - `差异类型`
   - `匹配字段`
   - `对比字段`
3. 详情中不出现“左侧”“右侧”作为数据列名。
4. 详情保留对账关键字段、差异字段和值、源数据集原始记录。
5. 旧数据缺少结构化详情时仍可用原摘要兜底展示。
