# 手机端查看差异页适配设计

## 背景

钉钉等通知通道里的“查看全量差异”链接会打开公开差异页：

- 前端页面：`finance-web/src/components/PublicReconRunExceptionsPage.tsx`
- API 路径：`/api/recon/public/runs/{run_id}/exceptions`
- 工作台内相近入口：`/api/recon/runs/{run_id}/exceptions`

PC 端当前表格布局已经适合批量扫描，不作为本次改造范围。手机端虽然可以打开页面，但核心差异列表仍是桌面宽表格：

- 差异列表容器使用 `min-w-[1320px]`，手机 390px 视口只能看到摘要列的一部分。
- 责任人、处理状态、操作按钮在横向滚动区域外，钉钉内查看成本高。
- 详情内“差异字段和值”使用 `min-w-[720px]` 表格，手机上仍需横向拖动。
- 原始记录区已有单列退化能力，基本可复用。

## 目标

在不改变 PC 端体验的前提下，让手机端满足通知场景里的快速查看：

1. 用户从钉钉打开链接后，能在手机首屏快速理解本次对账、日期和待处理差异数。
2. 差异列表在手机上不横向滚动，每条差异以卡片展示。
3. 用户能直接看到订单号、缺失侧或差异字段、责任人、处理状态。
4. 点击详情后能查看关键字段、差异值、左右原始记录的全部字段。
5. 保留现有 PC 端表格、筛选、分页、详情结构，不做桌面端视觉重做。

## 非目标

本次不扩展手机端处理闭环：

- 不新增责任人处理、状态更新、反馈录入入口。
- 不调整通知消息生成逻辑。
- 不改异常数据结构和后端 API。
- 不重构 `ReconWorkspace.tsx` 的工作台异常 modal。

## 推荐方案

采用“PC 表格 + 手机卡片流”的响应式双布局。

在 `md` 及以上视口继续渲染当前宽表格；在 `md` 以下隐藏桌面表格，渲染移动卡片列表。详情同理：桌面端继续使用当前居中 modal，手机端改为全屏详情层。

这种方案的优点是：

- PC 端风险最低，不触碰已有高密度表格体验。
- 手机端可以按阅读顺序重新组织信息，不被表格列宽约束。
- 大部分数据整理逻辑可复用现有 `businessDisplayForException`、`fieldValueSummary`、`PublicRecordSection`。

## 手机端页面结构

### 顶部摘要

手机端保留当前标题、状态、对账日期和运行指标，但视觉上压缩为纵向信息区：

- 标题：方案或任务名称。
- 状态：成功/失败/运行中 badge。
- 次级信息：对账日期。
- 指标：差异数、采集行数、整理后行数、对账耗时，以小卡片纵向或横向可滚动排列。
- “运行详情”继续折叠展示，避免首屏过长。

### 筛选区

手机端保留两项核心筛选：

- 处理状态下拉。
- 搜索框，提示文案保持“搜索订单号、金额、责任人”。

布局规则：

- 手机端纵向或两行排列，控件宽度占满。
- PC 端继续使用当前横向布局。

### 差异列表

手机端每条差异渲染为卡片，字段顺序如下：

1. 短摘要：`businessDisplay.shortSummary`
2. 关键字段：`fieldValueSummary(businessDisplay)`
3. 差异值摘要：取 `businessDisplay.compareLines[0]`，展示字段名、左侧值、右侧值或缺失侧。
4. 元信息：责任人、处理状态。
5. 操作：`查看详情` 按钮。

卡片规则：

- 单列列表，不横向滚动。
- 长订单号允许换行或 anywhere wrap。
- 摘要最多可自然换行，不做强截断，避免丢失订单号。
- `查看详情` 放在卡片底部右侧，保证拇指可点。

桌面端继续渲染当前 `min-w-[1320px]` 表格。

## 手机端详情层

手机端详情从居中 modal 调整为全屏层：

- 覆盖全屏，背景为 `bg-surface`。
- 顶部固定标题栏，包含“差异详情”和关闭按钮。
- 内容区独立纵向滚动。
- 底部不新增操作栏。

内容顺序：

1. 摘要与处理信息
   - 差异结论
   - 责任人
   - 处理状态
   - 最新反馈
2. 对账关键字段
   - 继续使用现有 `keyLines`
   - 手机端单列卡片
3. 差异字段和值
   - 手机端不使用表格
   - 每个 `compareLine` 是一个卡片
   - 展示字段名、左侧数据集值、右侧数据集值、差异值
4. 原始记录
   - 继续复用 `PublicRecordSection`
   - 手机端单列显示所有字段

桌面端继续使用当前 modal 和差异字段表格。

## 组件边界

为避免 `PublicReconRunExceptionsPage.tsx` 继续膨胀，建议拆出小组件：

- `MobileExceptionCard`
  - 输入：异常项、业务展示对象、责任人名称、处理状态。
  - 输出：手机端列表卡片。
- `ResponsiveCompareValues`
  - 输入：`compareLines`。
  - 桌面端渲染现有表格，手机端渲染卡片。
- `ResponsiveExceptionDetailDialog`
  - 包装现有详情内容。
  - 桌面端 modal，手机端全屏详情层。

如果实现时改动很小，也可以先把 `MobileExceptionCard` 单独抽出，详情层留在原文件内，但不得让 PC 端表格逻辑和手机卡片逻辑混杂成难读的大块条件渲染。

## 数据流

不改现有数据流：

1. 页面调用公开异常 bundle API。
2. `mapBundle` 归一化 run、scheme、exceptions。
3. `buildDisplayContext` 从 scheme meta 生成数据集名称和字段标签。
4. `businessDisplayForException` 生成摘要、关键字段、差异字段、原始记录分区。
5. 桌面端和手机端只在渲染层选择不同组件。

## 状态与错误处理

沿用现有行为：

- 加载中：显示“正在加载全量差异...”。
- 接口失败：显示当前错误提示。
- 无结果：显示“当前筛选条件下没有差异”。
- 分页：保留上一页/下一页。

手机端额外要求：

- 加载、错误、空结果状态必须在 390px 宽度下完整可读。
- 分页按钮在手机上可换行，但不能溢出。

## 响应式规则

建议断点：

- `< md`：手机布局。
- `>= md`：现有桌面布局。

关键规则：

- 桌面差异表格：`hidden md:block` 或等价结构。
- 手机差异卡片：`md:hidden`。
- 详情差异字段表格：桌面展示表格，手机展示卡片。
- 详情容器：桌面 `max-w-5xl rounded-[28px]`，手机 `fixed inset-0 rounded-none max-h-none`。

## 验收标准

手机端 390px 宽度下：

1. 页面无整体横向滚动。
2. 差异列表首屏能看到至少一条完整差异卡片的摘要、关键字段、责任人/状态和详情按钮。
3. 点击详情后，差异字段和值不出现横向表格。
4. 详情中原始记录字段单列展示，长字段值可换行。
5. 订单号 `5118001236570006333` 可在列表或详情中完整定位。
6. PC 端 1440px 宽度下差异表格仍按当前列布局展示。

## 测试计划

### 单元/组件测试

- 验证公开 bundle 加载后，移动卡片内容包含异常摘要、关键字段、责任人、处理状态、详情按钮。
- 验证点击详情后，原始记录字段在详情中出现。
- 验证 PC 表格相关文本仍存在，不破坏现有断言。

## 发布与回滚

该改造只影响前端渲染层。若手机端出现问题，可通过移除移动专用组件调用，恢复为当前桌面表格横向滚动行为。PC 端布局不作为回滚风险点，因为不做结构性修改。
