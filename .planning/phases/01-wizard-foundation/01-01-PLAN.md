<threat_model>
## Trust Boundaries

- Browser wizard state vs. user-visible stale/reference previews
- Frontend payload adapters vs. existing `/recon/schemes/design/*` session APIs
- Frontend save payload vs. final `/recon/schemes` validation gate

## STRIDE Register

| threat_id | category | component | disposition | mitigation_plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | Wizard derived artifact state | mitigate | Centralize invalidation in `updateIntentDraft`, `updatePreparationDraft`, and `updateReconciliationDraft` so stale generated JSON and trial results cannot remain marked current after upstream edits. |
| T-01-02 | Repudiation | Trial preview reference handling | mitigate | Preserve old previews only with explicit stale/reference metadata in the `derived` slice so the UI can label them `仅供参考` instead of silently mixing versions. |
| T-01-03 | Information Disclosure | Legacy payload adapter | accept | The plan keeps the current payload shape only inside the existing create-scheme flow and does not expose new technical fields to the business-facing UI. |
</threat_model>
