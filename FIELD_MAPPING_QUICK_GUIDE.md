### 场景2：修改一个文件的字段映射

#### 2.1 修改订单号字段

**原来的**：
```
业务数据的订单号 → "订单编号"
```

**需要修改为**：
```
业务数据的订单号 → "order_id"
```

**您应该输入**：
```
文件1的订单号改为"order_id"
或
业务文件的order_id改为"order_id"
```

**系统会**：
```
✏️ 文件1（业务数据） 修改 order_id: order_id
```

---

#### 2.2 修改金额字段

**您应该输入**：
```
文件2的金额改为"amount_in_cents"
或
财务数据的amount改为"amount_in_cents"
```

**系统会**：
```
✏️ 文件2（财务数据） 修改 amount: amount_in_cents
```

### 场景5：复杂操作（多个字段，多个文件）

#### 5.1 现实场景：Tencent异业对账

**示例背景**：
- 文件1：Tencent业务系统导出（列名：channel_order_id, revenue_amount, transaction_date）
- 文件2：财务系统导出（列名：order_no, income_money, trade_date）

**不匹配情况**：
- 订单号字段名不同
- 金额字段名不同（且revenue_amount需要除以100转为元）
- 日期字段名不同

**您应该输入（一次性处理所有调整）**：
```
文件1: 订单号改为"channel_order_id"，金额改为"revenue_amount"，日期改为"transaction_date"
文件2: 订单号改为"order_no"，金额改为"income_money"，日期改为"trade_date"
```

或更简洁的方式：
```
订单号：文件1对应channel_order_id，文件2对应order_no
金额：文件1对应revenue_amount，文件2对应income_money  
日期：文件1对应transaction_date，文件2对应trade_date
```

**系统会产生以下操作**：
```
✏️ 文件1（业务数据） 修改 order_id: channel_order_id
✏️ 文件1（业务数据） 修改 amount: revenue_amount
✏️ 文件1（业务数据） 修改 date: transaction_date
✏️ 文件2（财务数据） 修改 order_id: order_no
✏️ 文件2（财务数据） 修改 amount: income_money
✏️ 文件2（财务数据） 修改 date: trade_date
```

## ⚠️ 注意事项

### 不能删除的字段

```
❌ 删除订单号 (order_id是必需的)
❌ 删除金额 (amount是必需的)
❌ 删除日期 (date是必需的)
✅ 可以删除状态 (status是可选的)
```

系统会自动跳过这些操作。如果您尝试删除必需字段，系统会忽略该操作。

### 字段名规范

系统识别的字段角色（roles）：
- `order_id` - 订单号、交易号、单号等（用于匹配）
- `amount` - 金额、金额数值、收入等（用于金额校验）
- `date` - 日期、时间、交易日期等（用于日期对齐）
- `status` - 状态、订单状态等（可选，用于状态核对）

您应该根据这些角色来描述需要修改的字段，而不是具体的列名。

### 文件识别

系统识别两个文件的方式：
- 文件1 = 业务数据 = 第一个上传的文件
- 文件2 = 财务数据 = 第二个上传的文件

您可以使用以下任何表述：
- "文件1" 或 "文件2"
- "业务数据" 或 "财务数据"  
- "业务文件" 或 "财务文件"
- "第一个文件" 或 "第二个文件"

## 故障排查

### Q: 系统说"无法自动解析"

```
⚠️ 已记录你的调整意见，但未能自动解析。请详细描述需要修改的地方：
```

**解决方案**：
- 使用更清晰的表述
- 明确指出是修改(改为)、添加(添加)还是删除(删除)
- 确保文件名称正确("文件1"、"文件2"、"业务数据"或"财务数据")

**重试示例**：
```
❌ 不清楚: "订单号不对"
✅ 清楚: "文件1的订单号改为order_id"
```

### Q: 为什么无法删除某个字段

系统保护必需字段：
- 不支持删除：`order_id`、`amount`、`date`
- 可以删除：`status`

如果您尝试删除必需字段，系统会自动忽略该操作。

### Q: 如何知道修改是否生效

系统会显示操作摘要：
```
✏️ 文件1（业务数据） 修改 order_id: new_value
```

表示修改已成功。如果没有该行，说明操作未被执行或被跳过。
