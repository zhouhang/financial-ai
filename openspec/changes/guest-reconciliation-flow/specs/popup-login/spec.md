# Popup Login - 弹窗登录

## Overview

本规范定义了使用页面弹窗替代AI表单登录的交互流程。

## ADDED Requirements

### Requirement: 未登录用户显示登录按钮

系统 SHALL 在用户未登录时，在页面右上角显示登录按钮。

#### Scenario: 未登录用户访问
- **WHEN** 用户访问系统且未登录
- **THEN** 页面右上角显示"登录"按钮
- **AND** 按钮位置固定，不随页面滚动

#### Scenario: 已登录用户访问
- **WHEN** 用户访问系统且已登录
- **THEN** 顶部仅显示Logo（用户名和退出按钮在左侧Sidebar底部显示）
- **AND** 不显示登录按钮

### Requirement: 点击登录按钮打开弹窗

系统 SHALL 在用户点击登录按钮时，打开登录弹窗。

#### Scenario: 点击登录按钮
- **WHEN** 用户点击右上角"登录"按钮
- **THEN** 页面中央弹出登录框
- **AND** 弹窗遮罩背景，阻止点击其他区域

#### Scenario: 弹窗打开时
- **WHEN** 登录弹窗打开
- **THEN** 弹窗显示用户名输入框、密码输入框、登录按钮
- **AND** 弹窗显示"没有账号？注册"链接

### Requirement: 登录成功后自动保存推荐规则

系统 SHALL 在用户登录成功后，仅当用户此前已触发"是否保存推荐规则"提示并选择保存时，才自动将推荐规则保存到用户账户。

#### Scenario: 在保存规则提示下登录成功
- **WHEN** 用户在未登录状态下使用推荐规则对账，系统提示是否保存推荐规则，用户点击"保存"后，再点击登录
- **THEN** 登录成功后，系统将暂存的推荐规则保存到用户账户
- **AND** 刷新当前页面
- **AND** 关闭登录弹窗
- **AND** 更新左侧Sidebar用户信息

#### Scenario: 登录成功但无保存规则上下文
- **WHEN** 用户直接点击登录（未触发过保存规则提示）
- **THEN** 登录成功后仅刷新页面
- **AND** 不执行任何规则保存操作

#### Scenario: 登录成功但用户未选择保存
- **WHEN** 用户在保存规则提示下点击"不保存"后再登录
- **THEN** 登录成功后仅刷新页面
- **AND** 不执行任何规则保存操作

### Requirement: 登录失败显示错误

系统 SHALL 在登录失败时显示错误提示。

#### Scenario: 登录失败
- **WHEN** 用户输入错误的用户名或密码
- **THEN** 弹窗显示错误提示：用户名或密码错误
- **AND** 用户可重新输入

### Requirement: 关闭登录弹窗

系统 SHALL 允许用户关闭登录弹窗。

#### Scenario: 点击关闭按钮
- **WHEN** 用户点击弹窗关闭按钮（X）
- **THEN** 弹窗关闭
- **AND** 不执行任何登录操作

#### Scenario: 点击遮罩背景
- **WHEN** 用户点击弹窗遮罩背景
- **THEN** 弹窗关闭
- **AND** 不执行任何登录操作

└─────────────────────────────────────────┘

│  Logo                                  │  ← 已登录状态（用户信息在左侧Sidebar底部）

## 前端组件

### LoginModal 组件

| 属性 | 类型 | 说明 |
|------|------|------|
| isOpen | boolean | 是否打开弹窗 |
| onClose | () => void | 关闭弹窗回调 |
| onLoginSuccess | (user) => void | 登录成功回调 |

### 状态管理

- `isLoggedIn`: boolean - 是否已登录
- `currentUser`: User | null - 当前用户信息（显示在左侧Sidebar底部）
- `pendingSaveRules`: { ruleId: string, action: 'save' | 'discard' } | null - 用户在保存规则提示下的选择

## 行为变化

### 移除 AI 表单登录

- **移除**: AI 回复中的登录表单生成逻辑
- **移除**: `auth_handler` 中处理登录表单的逻辑
- **保留**: WebSocket 登录接口（弹窗登录复用）

### 保存推荐规则时机

1. 用户在未登录状态下使用推荐规则对账 → 系统提示是否保存推荐规则
2. 用户选择"保存" → 记录 pendingSaveRules = { ruleId, action: 'save' }
3. 用户点击登录 → 登录成功
4. 检查 pendingSaveRules，如果 action 是 'save' 则保存规则 → 刷新页面
