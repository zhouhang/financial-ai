## Why

用户在对账时需要手动创建规则，当规则库中存在大量规则（几万甚至几十万条）时，寻找合适的规则非常耗时。现有功能仅支持字段映射建议，但缺乏基于现有规则库智能推荐匹配规则的能力，导致用户操作效率低下。

## What Changes

- 上传文件后，在现有 LLM 建议字段映射的基础上，增加规则推荐功能
- 根据文件表头字段，查询现有规则库，匹配订单号、金额、订单时间等关键字段，智能推荐最可能的规则
- 查询需考虑性能优化，支持大规模规则库的高效检索
- 用户可直接使用推荐规则进行对账，无需手动选择
- 对账完成后，AI 给出规则是否适合作为长期使用规则的结论，并提示用户保存
- 用户确认后，可将规则保存为个人规则

## Capabilities

### New Capabilities

- `rule-recommendation`: 智能规则推荐功能
  - 文件上传后分析表头字段
  - 查询现有规则库，匹配关键字段（订单号、金额、订单时间）
  - 返回推荐规则列表及匹配度说明
  - 支持直接使用推荐规则进行对账
- `rule-evaluation`: 对账结果评估功能
  - 对账完成后分析规则适用性
  - 给出规则是否适合作为长期规则的结论
  - 提供保存规则为个人规则的交互

### Modified Capabilities

- (无)

## Impact

- **前端** (finance-web): 新增规则推荐 UI 组件、推荐结果展示、对账结果评估界面、保存规则交互
- **后端** (finance-mcp): 新增规则推荐 API、智能匹配算法、对账结果评估 API
- **数据层**: 规则表查询优化，可能需要添加索引或缓存策略
- **外部依赖**: 无新增依赖
